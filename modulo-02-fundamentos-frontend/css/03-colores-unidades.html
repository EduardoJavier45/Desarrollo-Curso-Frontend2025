🎨 Colores y Unidades en CSS

Aprende a dar estilo con colores y medidas

🌈 Formatos de Colores en CSS

CSS ofrece múltiples formas de definir colores. Cada una tiene sus ventajas:

1. Nombres de Colores (140 predefinidos)

RED
color: red;
BLUE
color: blue;
GREEN
color: green;
ORANGE
color: orange;
PURPLE
color: purple;
TEAL
color: teal;
💡 Ventaja: Fáciles de recordar y escribir.
⚠️ Desventaja: Solo 140 colores disponibles, limitado para diseños personalizados.

2. Hexadecimal (#RRGGBB)

El formato más popular. Usa números hexadecimales (0-9, A-F) para definir Rojo-Verde-Azul:

#FF6B6B
color: #FF6B6B;
#4ECDC4
color: #4ECDC4;
#45B7D1
color: #45B7D1;
#F38181
color: #F38181;
#AA96DA
color: #AA96DA;
#FCBAD3
color: #FCBAD3;
/* Formato completo: #RRGGBB */
color: #FF6B6B;  /* RR=FF (rojo), GG=6B (verde), BB=6B (azul) */

/* Formato corto: #RGB (se duplica cada dígito) */
color: #F00;     /* Equivalente a #FF0000 */
color: #0F0;     /* Equivalente a #00FF00 */
💡 Ventaja: 16.7 millones de colores disponibles, formato estándar.
⚠️ Desventaja: No soporta transparencia (usar #RRGGBBAA en CSS moderno).

3. RGB - Red, Green, Blue

Define colores con valores de 0 a 255 para cada canal:

RGB(255, 107, 107)
rgb(255, 107, 107)
RGB(78, 205, 196)
rgb(78, 205, 196)
RGB(69, 183, 209)
rgb(69, 183, 209)
color: rgb(255, 107, 107);  /* Rojo=255, Verde=107, Azul=107 */
color: rgb(0, 0, 0);        /* Negro */
color: rgb(255, 255, 255);  /* Blanco */

4. RGBA - RGB con Alpha (Transparencia)

Igual que RGB pero con un cuarto valor (0.0 a 1.0) para la opacidad:

RGBA 80%
rgba(255, 107, 107, 0.8)
RGBA 60%
rgba(78, 205, 196, 0.6)
RGBA 40%
rgba(69, 183, 209, 0.4)
background: rgba(255, 107, 107, 0.8);  /* 80% opaco */
background: rgba(0, 0, 0, 0.5);        /* Negro semi-transparente */
background: rgba(255, 255, 255, 0.1);  /* Blanco casi transparente */
💡 Ventaja: Permite transparencias, ideal para overlays.
✅ Uso común: Fondos semi-transparentes, sombras, capas.

5. HSL - Hue, Saturation, Lightness

Más intuitivo para diseñadores: Tono (0-360°), Saturación (0-100%), Luminosidad (0-100%)

HSL(0, 100%, 71%)
hsl(0, 100%, 71%)
HSL(174, 59%, 55%)
hsl(174, 59%, 55%)
HSL(194, 59%, 54%)
hsl(194, 59%, 54%)
/* Tono (Hue): 0=rojo, 120=verde, 240=azul */
color: hsl(0, 100%, 50%);    /* Rojo puro */
color: hsl(120, 100%, 50%);  /* Verde puro */
color: hsl(240, 100%, 50%);  /* Azul puro */

/* Saturación: 0%=gris, 100%=color puro */
color: hsl(0, 0%, 50%);      /* Gris */
color: hsl(0, 50%, 50%);     /* Rojo desaturado */
color: hsl(0, 100%, 50%);    /* Rojo saturado */

/* Luminosidad: 0%=negro, 50%=normal, 100%=blanco */
color: hsl(0, 100%, 0%);     /* Negro */
color: hsl(0, 100%, 50%);    /* Rojo normal */
color: hsl(0, 100%, 100%);   /* Blanco */

6. HSLA - HSL con Alpha

HSLA 70%
hsla(0, 100%, 71%, 0.7)
HSLA 70%
hsla(174, 59%, 55%, 0.7)
💡 Ventaja HSL: Fácil crear variaciones (más claro/oscuro, más/menos saturado).
✅ Ideal para: Temas de color, paletas, variaciones automáticas.

📏 Unidades de Medida en CSS

📌 Unidades Absolutas (tamaño fijo)

Píxeles (px)

Este texto tiene 16px de tamaño
font-size: 16px;
width: 300px;
padding: 20px;
✅ Uso: Cuando necesitas precisión exacta.
⚠️ Problema: No escala con las preferencias del usuario.

Puntos (pt)

Este texto tiene 12pt de tamaño (para impresión) 
font-size: 12pt;  /* 1pt = 1/72 de pulgada */
✅ Uso: Estilos de impresión (@media print).
❌ No usar: Para pantallas (usar px, rem, em).

Centímetros (cm)

Esta caja tiene 5cm de ancho (en impresión) 
width: 5cm;  /* Útil para impresión */
Unidad Nombre Uso
px Píxeles General, web
pt Puntos Impresión
cm Centímetros Impresión
mm Milímetros Impresión
in Pulgadas Impresión

🔄 Unidades Relativas (adaptan tamaño)

EM - Relativo al padre

Padre: 16px (font-size base)
Hijo: 1.5em = 24px (1.5 × 16px)
Nieto: 1.5em = 36px (1.5 × 24px) 
.parent { font-size: 16px; }
.child { font-size: 1.5em; }      /* 1.5 × 16 = 24px */
.grandchild { font-size: 1.5em; } /* 1.5 × 24 = 36px */
⚠️ Problema: Efecto "multiplicador" en elementos anidados.
✅ Solución: Usar REM en su lugar.

REM - Relativo al root (html)

1rem = 16px (tamaño base)
1.5rem = 24px
2rem = 32px
html { font-size: 16px; }  /* Base */

.small { font-size: 1rem; }    /* 16px */
.medium { font-size: 1.5rem; } /* 24px */
.large { font-size: 2rem; }    /* 32px */
✅ Ventaja: Siempre relativo al tamaño base, sin efecto multiplicador.
🏆 Mejor práctica: Usar REM para tamaños de fuente y espaciado.

Porcentaje (%)

Contenedor: 100%
Hijo: 50%
Hijo: 75%
.container { width: 100%; }
.half { width: 50%; }       /* 50% del padre */
.three-quarters { width: 75%; }
✅ Uso: Layouts fluidos, diseño responsivo.
💡 Nota: Siempre relativo al elemento padre.

Unidades de Viewport

50vw = 50% del ancho del navegador
20vh = 20% de la altura del navegador
30vmin × 30vmin
(círculo responsivo) 
/* VW - Viewport Width (ancho) */
width: 50vw;   /* 50% del ancho de la ventana */

/* VH - Viewport Height (alto) */
height: 100vh; /* 100% de la altura de la ventana */

/* VMIN - El menor entre vw y vh */
width: 30vmin;  /* 30% de la dimensión más pequeña */

/* VMAX - El mayor entre vw y vh */
width: 30vmax;  /* 30% de la dimensión más grande */
✅ Uso común:
• 100vh para secciones de altura completa
• vw para tipografía responsiva
• vmin/vmax para elementos cuadrados adaptables
Unidad Relativo a Uso recomendado
em Tamaño de fuente del padre Espaciado interno de componentes
rem Tamaño de fuente del root (html) 🏆 Tamaños de fuente, márgenes globales
% Elemento padre Layouts, anchos de columnas
vw 1% del ancho del viewport Anchos responsivos, tipografía fluida
vh 1% de la altura del viewport Secciones de altura completa
vmin 1% de la dimensión menor Elementos cuadrados
vmax 1% de la dimensión mayor Fondos, overlays

✅ Mejores Prácticas

Colores:

  • Usa hexadecimal para colores sólidos simples
  • Usa RGBA/HSLA cuando necesites transparencia
  • Usa HSL para crear paletas y variaciones de color
  • Define colores en variables CSS (--color-primary)
  • Asegura suficiente contraste para accesibilidad (WCAG 4.5:1)

Unidades:

  • Usa rem para tamaños de fuente (accesibilidad)
  • Usa px para bordes y sombras (precisión)
  • Usa % para anchos de layouts
  • Usa vh/vw para secciones full-screen
  • Evita pt, cm, mm excepto para impresión
  • Combina unidades: max-width: 1200px con width: 90% 
/* ✅ Buenas prácticas combinadas */
:root {
    --color-primary: #667eea;
    --color-secondary: #764ba2;
    --spacing-unit: 1rem;
}

body {
    font-size: 16px;        /* Base para rem */
    color: #333;            /* Hexadecimal para sólido */
    background: rgba(102, 126, 234, 0.1); /* RGBA para transparencia */
}

.container {
    max-width: 1200px;      /* px para límite */
    width: 90%;             /* % para responsive */
    margin: 0 auto;
    padding: var(--spacing-unit);
}

h1 {
    font-size: 2.5rem;      /* rem para escalable */
    margin-bottom: 1rem;
}

.hero {
    height: 100vh;          /* vh para full-screen */
    background: linear-gradient(135deg, 
        var(--color-primary) 0%, 
        var(--color-secondary) 100%);
}
