⚖️ Especificidad en CSS

Entendiendo cómo CSS decide qué estilos aplicar

¿Qué es la Especificidad?

La especificidad es el mecanismo que usa CSS para determinar qué regla se aplica cuando múltiples reglas afectan al mismo elemento.

Piensa en ella como un "peso" o "puntuación" que se calcula para cada selector. El selector con mayor especificidad gana.

Regla de oro: Cuanto más específico es un selector, mayor prioridad tiene.

Cálculo de Especificidad

La especificidad se representa con 4 números: (a, b, c, d)

Componente Valor Descripción Ejemplo
a 1000 Estilos en línea style="color: red;"
b 100 IDs #principal
c 10 Clases, atributos, pseudo-clases .menu, [type="text"], :hover
d 1 Elementos, pseudo-elementos p, div, ::before

Ejemplos Prácticos de Especificidad

Ejemplo 1: Elemento vs Clase vs ID

/* Especificidad: 0,0,0,1 = 1 */
p { color: black; }

/* Especificidad: 0,0,1,0 = 10 */
.texto { color: blue; }

/* Especificidad: 0,1,0,0 = 100 */
#parrafo-especial { color: red; }

/* Especificidad: 0,0,1,0 = 10 */
.caja { background-color: lightgray; }

Ejemplo 3: Orden de Aparición (Empate)

/* Ambas tienen especificidad: 0,0,1,0 = 10 */
.color-prueba { color: blue; }
.color-prueba { color: green; } /* Esta gana (última) */

Este texto será verde (segunda regla gana)

Ejemplo 4: !important (El Tramposo)

.normal { color: blue; }
#texto-forzado { color: green; }
.forzado { color: red !important; } /* GANA SIEMPRE */

Texto azul (clase normal)

Texto rojo (¡important gana sobre ID!)

⚠️ ADVERTENCIA: Evita usar !important. Rompe la cascada natural y dificulta el mantenimiento.

Reglas y Mejores Prácticas

✅ Buenas Prácticas:

  • Usa clases para estilos reutilizables
  • Reserva IDs para JavaScript y anclas
  • Mantén la especificidad baja (más fácil de sobrescribir)
  • Evita selectores demasiado anidados (div > ul > li > a)
  • Usa metodologías como BEM para organizar clases
  • Prefiere especificidad por clases múltiples sobre IDs

❌ Malas Prácticas:

  • Usar !important para "solucionar" problemas
  • Selectores excesivamente específicos
  • Mezclar estilos en línea con CSS externo
  • Usar IDs para estilos (muy alta especificidad)
  • No entender por qué un estilo no se aplica
💡 Consejo profesional: Si necesitas sobrescribir un estilo, aumenta la especificidad de forma lógica en lugar de usar !important.

Orden de Prioridad (Cascada Completa)

Cuando hay conflicto, CSS decide en este orden:

  1. Importancia: !important del autor > normal
  2. Especificidad: ID (100) > Clase (10) > Elemento (1)
  3. Orden de aparición: La última regla gana (en empates)
  4. Herencia: Propiedades heredadas de padres

Ejemplo completo:

/* 1. Específico pero primero */
#texto { color: blue; }          /* 0,1,0,0 = 100 */

/* 2. Menos específico */
.parrafo { color: green; }       /* 0,0,1,0 = 10 */

/* 3. Más específico - GANA */
#texto.parrafo { color: red; }   /* 0,1,1,0 = 110 */

/* 4. Máxima prioridad - GANA SOBRE TODO */
.parrafo { color: yellow !important; }

El texto será amarillo (por !important)
